## What changed
- expanded `src/security.js` so `evaluateSecurityPosture()` scans prompt plus other text-bearing change-request fields like title, summary, rationale, notes, validation checks, and next actions
- added field-aware prompt-injection evidence, including `surfacesScanned` and `flaggedSurfaces`, so review routing is easier to audit locally
- hardened `src/proposal.js` so destructive, restart, secret-touching, and external-effect flags clamp proposals upward to stricter risk classes instead of silently staying too permissive
- added optional proposal risk-floor preservation via `securityPosture.riskClass`
- clamped proposal approval boundaries so a supplied boundary cannot be weaker than the implied risk class
- added `examples/security-posture-non-prompt-injection.example.json` plus a matching npm script for manual verification
- expanded `src/self-test.js` to cover:
  - risky-flag proposal clamping
  - proposal risk floors from prior security posture
  - non-prompt summary detection
  - non-prompt list-field detection
  - CLI manifest coverage
  - CLI verification of the new non-prompt injection example
- updated `README.md`, `docs/USAGE.md`, `docs/SECURITY-MODEL.md`, and `docs/PLUGIN-CONTRACT.md` so the documented security surface matches the new behavior
