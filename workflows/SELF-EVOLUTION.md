# Self-Evolution Workflow

## Goal
Improve both the assistant and the operating system around it.

## Scope
The workflow may improve:
- memory rules
- style-fit files
- cron schedules
- workflow prompts
- report cadence
- maintenance routines

## Phases
1. Research
2. Evaluate
3. Implement
4. Reflect

## Evaluation rubric
- user-fit gain
- friction reduction
- safety / rollback ease
- disruption risk
- schedule quality
- prompt quality
- memory quality

## Output
- daily evolution report
- changes implemented
- proposed next moves
